# Incident Response Report Project

## Project Overview
This project documents a simulated incident response investigation involving a suspected malware infection and command-and-control activity. The aim was to practise identifying indicators of compromise, following the incident response lifecycle, and documenting containment, eradication, recovery, and lessons learned.

## Scenario
A simulated endpoint alert was triggered after a workstation showed unusual outbound connections, suspicious process activity, and signs of possible malware execution. The incident was reviewed from a SOC analyst perspective and documented in an incident response report.

## Skills Practised
- Incident response documentation
- Malware indicator review
- Timeline analysis
- Indicator of Compromise identification
- Containment planning
- Eradication and recovery recommendations
- Lessons learned reporting
